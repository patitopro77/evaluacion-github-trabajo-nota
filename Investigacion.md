# Investigación Complementaria: Git y GitHub

1. ¿Qué es Git?
Es un sistema de control de versiones distribuido que rastrea los cambios en el código fuente durante el desarrollo de software.

2. ¿Qué es GitHub?
Es una plataforma en la nube que hospeda proyectos utilizando el sistema Git, facilitando el trabajo colaborativo.

3. ¿Cuál es la diferencia entre Git y GitHub?
Git es la herramienta local de control de versiones que instalas en tu computadora, mientras que GitHub es el servicio en línea para almacenar y compartir esos proyectos.

4. ¿Qué es un repositorio?
Es una carpeta o espacio de almacenamiento virtual donde se guardan todos los archivos de un proyecto y su historial de cambios.

5. ¿Qué función cumple git add?
Agrega los cambios de los archivos modificados al área de preparación (staging area) para incluirlos en el próximo envío.

6. ¿Qué función cumple git commit?
Guarda permanentemente los cambios preparados en el historial local del repositorio, creando un punto de restauración con un mensaje descriptivo.

7. ¿Qué función cumple git push?
Sube los commits realizados en el repositorio local hacia un repositorio remoto en la nube (como GitHub).

8. ¿Qué función cumple git pull?
Descarga e integra los últimos cambios del repositorio remoto directamente en el repositorio local para mantenerlo actualizado.

9. ¿Cuáles son las ventajas del control de versiones?
* Permite regresar a versiones anteriores del código.
* Facilita el trabajo en equipo sin sobrescribir archivos.
* Mantiene un historial detallado de quién hizo cada cambio.
* Permite experimentar en ramas separadas sin alterar el proyecto principal.
